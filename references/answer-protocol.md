## 结构化代答协议

当运行环境可以直接和目标 persona skill 对话时，使用这份协议。

### 目标

让目标 skill 以“这个被蒸馏出来的人物本人”的视角回答 SBTI 问卷，不让用户亲自作答。

### 必须返回的格式

只返回 JSON，不要夹杂解释文本：

```json
{
  "answers": [
    {
      "id": "q1",
      "option": 2,
      "confidence": 0.68,
      "evidence": "我经常嘴上自嘲，也会提到自己普通又敏感。"
    }
  ]
}
```

### 字段规则

- `id` 必须和问卷题目的 id 完全一致
- `option` 必须是该题合法的数字选项值
- `confidence` 必须在 `0.0` 到 `1.0` 之间
- `evidence` 必须是基于目标 skill 的记忆或 persona 材料给出的简短依据
- 如果拿不准，也要继续答，但要降低 `confidence`
- 不要反问用户，不要要求用户补做题

### 建议的提问方式

1. 先告诉目标 skill：这是一份代理式 SBTI 测评
2. 明确要求它按“稳定人格倾向”作答，不按某一次情绪波动作答
3. 要求它在整套题里保持前后一致
4. 强调只返回 JSON
